/api/v1/items

There's items list in the api that can be added to the inventory when a survivor is found and registred.

The survivors can make item trades.

Remember, you can't add new items.

/api/v1/items get

get

Show all items registred in database.

get /api/v1/items

Show all items registred in database.

  • Response

HTTP status code 200

Should return json array with all items.

Body

Type: application/json

Example:

[
  {
    "id": 1,
    "description": "water",
    "points": 4
  },
  {
    "id": 2,
    "description": "medication",
    "points": 2
  },
  {
    "id": 3,
    "description": "ammunition",
    "points": 1
  },
  {
    "id": 4,
    "description": "food",
    "points": 3
  }
]

/api/v1/items/trade post

The items in survivors inventory can be trade.

post

Make items trade between two survivors.

post /api/v1/items/trade

Make items trade between two survivors.

  • Request

Body

Type: application/json

Example:

{
    "dealer_1": {
        "id": 1,
        "swap_items": [
            {
                "id": 1,
                "quantity" : 2
            },
            {
                "id" : 2,
                "quantity": 1
            }
        ]
    },
    "dealer_2": {
        "id": 2,
        "swap_items": [
            {
                "id": 3,
                "quantity" : 10
            }
        ]
    }
}

post /api/v1/survivors

Add new survivor with/without inventory. To know item_id you should GET api/v1/items

  • Request
  • Response

Body

Type: application/json

Example:

{
    "survivor" : {
        "name": "Jessica Jones",
        "age": 32,
        "gender": "F",
        "last_latitude": "66666",
        "last_longitude": "66666",
        "inventories_attributes": [
            {
                "quantity" : 33,
                "item_id": 1
            }
        ]
    }
}

HTTP status code 201

Survivor registred.

Headers

  • Location: (string)

    survivor path.

    Example:

    Location: /api/v1/survivros/{id}

Body

Type: application/json

Example:

{
  "id": 6,
  "name": "Jessica Jones",
  "age": 32,
  "gender": "F",
  "last_latitude": "66666",
  "last_longitude": "66666",
  "infected": false,
  "inventory_points": 132,
  "inventories": [
    {
      "id": 4,
      "quantity": 33,
      "item": {
        "id": 1,
        "description": "water",
        "points": 4
      }
    }
  ]
}

post

Report infection.

Report infection.

  • Request
  • Response

URI Parameters

  • id: required (string)

Body

Type: application/json

Example:

{
    "infected_id": "10"
}

Body

Type: application/json

Example:

{
  "message": "Your report has been registered."
}

/api/v1/reports

Api provides some reports. You can GET all reports or a specific report informing a type with query parameters.

/api/v1/reports get

get

Get all reports reports or specific.

get /api/v1/reports

Get all reports reports or specific.

  • Request
  • Response

Query Parameters

  • type: (string)

    If you want get a specific report, you should inform this param one or many of [ total_survivors, infected, not_infected, items_average_by_survivor, lost_points ].

    By default ( if you don't inform any type ), the api will return all reports.

    Example:

    /api/v1/reports?types=infected, not_infected

HTTP status code 200

Should return json array with reports.

Body

Type: application/json

Example:

[
  {
    "type": "infected",
    "data": {
      "total": 0,
      "percentage": 0,
      "survivors": []
    }
  },
  {

      "total": 6,
